## Введение
**Ollama Server** — это проект, который позволяет запустить службу Ollama на устройствах Android одним щелчком мыши. Не полагаясь на Termux, он позволяет пользователям легко запускать большие языковые модели на устройствах Android.

Служба Ollama, запускаемая **Ollama Server**, ничем не отличается от службы, запускаемой другими методами. Вы можете выбрать любой клиент, который вызывает Ollama для взаимодействия с API, предоставляемым сервисом Ollama.

> **Этот форк** обновляет встроенную цель ollama до **v0.23.2** и расширяет клиентский API современными функциями: мультимодальными изображениями, вызовом инструментов, встраиванием, параметрами чата (температура, top_p и т. д.) и рекомендациями по динамическим моделям.

## Особенности
- **Развертывание в один клик**: легко запускайте и управляйте службой Ollama.
- **Termux не требуется**: работает независимо, без дополнительной эмуляции терминала.

## Поддерживаемые возможности
- Запуск/остановка службы Ollama в один клик.
- Извлекайте модели из официальной библиотеки Олламы.
- Загружайте собственные модели `.gguf` с автоматически определяемыми шаблонами (Llama, Mistral, Gemma, ChatML)
- Удаление и выгрузка моделей
- Чат с потоковым рендерингом уценки
- История разговоров с резюме
- Переключатель локального/внешнего доступа
- Просмотр журнала сервера
- **Мультимодальный чат** — поддержка ввода изображений (модели видения)
- **Параметры чата** — температура, top_p, top_k, num_ctx и т. д.
- **Вызов инструмента** — определения функций/инструментов для рабочих процессов агента.
- **API встраивания** — поддержка `/api/embed`
- **Информация о модели** — `/api/show` для получения подробной информации о модели.
- **Рекомендации по динамическим моделям** – объединяет удаленный курируемый список с локально установленными моделями.
- Конфигурация **keep_alive** — контролирует, как долго модели остаются загруженными в памяти.
- Поддержка **эмулятора x86_64** — дополнительная цель сборки для эмулятора Android.

## Установка
1. Загрузите последнюю версию с [GitHub Releases] (https://github.com/theWatcherNineteen83/OllamaServer/releases).
2. Установите APK на свое устройство Android (arm64-v8a).
3. Откройте приложение и запустите сервис Ollama одним щелчком мыши.

## Системные требования

### Версия Android
| Требование | Значение |
|------------|-------|
| **Минимальная версия Android** | 9.0 (пирог, API 28) |
| **Целевой SDK** | 28 (обновлено с оригинальной версии; для Play Store требуется ≥34) |
| **Компилировать SDK** | 35 |

Требуется Android 9 (2018) или новее. Устройства под управлением Android 8 и более ранних версий **не поддерживаются**.

### Архитектура ЦП
| Архитектура | Поддерживается | Заметки |
|---|---|---|
| **arm64-v8a** | ✅ Да | Все современные 64-битные телефоны (2015+) |
| **armeabi-v7a** | ⚠️ Только сборка | Не включено в разделение APK; двоичный файл не входит в комплект. Требуется сборка NDK и изменение конфигурации разделения. |
| **x86_64** | ⚠️ Только для эмулятора | Для эмулятора Android. Используйте BUILD_X86=1 ./build_ollama_android.sh. |
| **x86** | ❌ Нет | Не поддерживается. |

> **На практике:** APK-файл работает практически на всех телефонах Android, начиная с 2018 года (Snapdragon 835 и новее, все 64-битные чипы ARM).

### ОЗУ (Память)
Требуемая память зависит в первую очередь от используемой вами модели, а не от самого приложения.

Затраты на приложение и сервер ollama составляют **~200–400 МБ**. Добавьте размер модели:

| Модель | Размер | Мин. Оперативная память устройства |
|-------|------|-----------------|
| qwen2.5:0.5b / qwen3:0.6b | ~400 МБ | **3 ГБ** |
| лама3.2:1b / гемма3:1b | ~0,8–1,3 ГБ | **4 ГБ** |
| qwen3:1.7b / фи4-мини:3.8b | ~1–2,2 ГБ | **6 ГБ** |
| лама3.2:3b | ~2 ГБ | **6 ГБ** |
| мистраль:7b | ~4,1 ГБ | **8 ГБ** |

**Рекомендация**: 6 ГБ ОЗУ или больше для комфортной работы с моделями 1B–3B. 8 ГБ для моделей 7B.

> ⚠️ Использование моделей с объемом оперативной памяти, близким к пределу вашего устройства, приведет к тому, что Android отключит службу или приложение.

### Другое
- **Хранилище:** ~2–5 ГБ бесплатно (для приложения, двоичных файлов ollama и файлов моделей).
- **Интернет:** Требуется только для загрузки моделей (по запросу)
- **ГП:** Не используется (только вывод ЦП)

### Примечания по обратной совместимости
- APK создан **только для Arm64-v8a**. Для добавления «armeabi-v7a» (32-разрядной версии) требуется скомпилировать двоичный файл ollama с «GOARCH=arm» в NDK и добавить «armeabi-v7a» в список «splits.abi.include» в «android/app/build.gradle».
- `targetSdkVersion 28` ниже минимального значения в Google Play Store (34). Чтобы опубликовать в Play Store, обновите targetSdkVersion до 34 в android/build.gradle.
- 32-разрядные устройства ARM (armeabi-v7a) обычно имеют ОЗУ ≤3 ГБ, что делает их непригодными для моделей, превышающих 0,5B.
- Устройства Android x86 (например, некоторые ASUS Zenfones, планшеты на базе процессоров Intel) не поддерживаются.

### Сборка из исходников
``` баш
# Предварительные требования: Node 18+, Android NDK r26+, Go 1.22+.
НПМ CI
Запуск выставки npx: Android

# Чтобы обновить прилагаемый двоичный файл ollama:
экспортировать ANDROID_NDK_HOME=/путь/к/ndk
./build_ollama_android.sh v0.23.2

# Необязательно: сборка x86_64 для эмулятора
BUILD_X86=1 ./build_ollama_android.sh v0.23.2
```

См. [BUILD_ANDROID.md](./BUILD_ANDROID.md) для получения подробных инструкций по двоичной компиляции.
