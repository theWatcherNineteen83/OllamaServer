## Введение
**Ollama Server** — это проект, позволяющий запускать сервис Ollama одним нажатием на устройствах Android. Не полагаясь на Termux, он позволяет пользователям легко запускать большие языковые модели на устройствах Android.

Сервис Ollama, запущенный с помощью **Ollama Server**, ничем не отличается от того, что запускается другими способами. Вы можете выбрать любой клиент, который вызывает Ollama, для взаимодействия с API, предоставляемым сервисом Ollama.

> **Этот форк** обновляет встроенную цель ollama до версии **v0.23.2** и расширяет клиентский API современными функциями: мультимодальные изображения, вызов инструментов, встраивание, параметры чата (temperature, top_p и т. д.) и динамические рекомендации моделей.

## Особенности
- **Развертывание в один клик**: легко запускайте и управляйте сервисом Ollama.
- **Termux не требуется**: Работает автономно без дополнительной эмуляции терминала.

## Поддерживаемые возможности
- Запуск/остановка сервиса Ollama в один клик
- Загрузка моделей из официальной библиотеки Ollama
- Загрузка пользовательских моделей `.gguf` с автоматически определяемыми шаблонами (Llama, Mistral, Gemma, ChatML)
- Удаление и выгрузка моделей
- Чат с потоковым рендерингом Markdown
- История разговоров с резюме
- Переключение между локальной сетью и внешним доступом
- Просмотр журнала сервера
- **Мультимодальный чат** — поддержка ввода изображений (модели зрительного восприятия)
- **Параметры чата** — temperature, top_p, top_k, num_ctx и другие
- **Вызов инструментов** — определения функций/инструментов для рабочих процессов агентов
- **API встраивания** — поддержка `/api/embed`
- **Информация о модели** — `/api/show` для подробностей о модели
- **Динамические рекомендации моделей** — объединяет удаленный курируемый список с локально установленными моделями
- Настройка **keep_alive** — управление временем, в течение которого модели остаются загруженными в памяти
- Поддержка **эмулятора x86_64** — опциональная цель сборки для эмулятора Android

## Установка
1. Загрузите последнюю версию с [GitHub Releases](https://github.com/theWatcherNineteen83/OllamaServer/releases).
2. Установите APK на ваше устройство Android (arm64-v8a).
3. Откройте приложение и запустите сервис Ollama одним нажатием.

## Системные требования

### Версия Android
| Требование | Значение |
|------------|-------|
| **Минимальная версия Android** | 9.0 (Pie, API 28) |
| **Целевой SDK** | 28 (обновлен по сравнению с исходным; Play Store требует ≥34) |
| **Компиляционный SDK** | 35 |

Требуется Android 9 (2018) или более поздней версии. Устройства под управлением Android 8 или более старых версий **не поддерживаются**.

### Архитектура процессора
| Архитектура | Поддерживается | Примечания |
|---|---|---|
| **arm64-v8a** | ✅ Да | Все современные 64-разрядные телефоны (2015+) |
| **armeabi-v7a** | ⚠️ Только для сборки | Не включено в разбиение APK; бинарный файл не входит в комплект. Требуется сборка NDK и изменение конфигурации разбиения. |
| **x86_64** | ⚠️ Только для эмулятора | Для эмулятора Android. Используйте `BUILD_X86=1 ./build_ollama_android.sh`. |
| **x86** | ❌ Нет | Не поддерживается. |

> **На практике:** APK работает практически на всех телефонах Android, выпущенных с 2018 года (Snapdragon 835 и новее, все 64-разрядные чипы ARM).

### ОЗУ (память)
Требования к памяти зависят в первую очередь от модели, которую вы запускаете, а не от самого приложения.

Нагрузка от приложения + сервера ollama составляет **~200–400 МБ**. Добавьте размер модели:

| Модель | Размер | Мин. ОЗУ устройства |
|-------|------|------------- ----|
| qwen2.5:0.5b / qwen3:0.6b | ~400 МБ | **3 ГБ** |
| llama3.2:1b / gemma3:1b | ~0,8–1,3 ГБ | **4 ГБ** |
| qwen3:1.7b / phi4-mini:3.8b | ~1–2,2 ГБ | **6 ГБ** |
| llama3.2:3b | ~2 ГБ | **6 ГБ** |
| mistral:7b | ~4,1 ГБ | **8 ГБ** |

**Рекомендация:** 6 ГБ ОЗУ или более для комфортной работы с моделями объемом 1–3 млрд. 8 ГБ для моделей объемом 7 млрд.

> ⚠️ Запуск моделей, занимающих объем, близкий к пределу ОЗУ вашего устройства, приведет к тому, что Android завершит работу службы или приложения.

### Прочее
- **Память:** ~2–5 ГБ свободного места (для приложения, бинарного файла ollama и файлов моделей)
- **Интернет:** Требуется только для загрузки моделей (pull)
- **GPU:** Не используется (только вычисления на CPU)

### Примечания по обратной совместимости
- APK скомпилирован **только для arm64-v8a**. Для добавления `armeabi-v7a` (32-битная версия) необходимо скомпилировать бинарный файл ollama с параметром `GOARCH=arm` в NDK и добавить `«armeabi-v7a»` в список `splits.abi.include` в файле `android/app/build.gradle`.
- `targetSdkVersion 28` ниже минимального требования Google Play Store (34). Чтобы опубликовать приложение в Play Store, обновите `targetSdkVersion` до `34` в `android/build.gradle`.
- 32-разрядные устройства ARM (armeabi-v7a) обычно имеют ≤3 ГБ ОЗУ, что делает их неподходящими для моделей объемом более 0,5 млрд.
- Устройства Android на базе x86 (например, некоторые ASUS Zenfone, планшеты на базе Intel) не поддерживаются.

### Сборка из исходного кода
```bash
# Требования: Node 18+, Android NDK r26+, Go 1.22+
npm ci
npx expo run:android

# Чтобы обновить входящий в комплект бинарник ollama:
export ANDROID_NDK_HOME=/path/to/ndk
./build_ollama_android.sh v0.23.2

# Дополнительно: сборка x86_64 для эмулятора
BUILD_X86=1 ./build_ollama_android.sh v0.23.2
```

См. [BUILD_ANDROID.md](./BUILD_ANDROID.md) для подробных инструкций по компиляции бинарного файла.
